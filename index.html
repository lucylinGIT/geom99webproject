PROJECT OVERVIEW EDITING OUR MAP DISPLAYING OUR MAP

The Problem

Ganaraska Region Conservation Authority plans to release web maps to their users for offline navigation of their Ganaraska Forest Trail network. Currently users rely on paper maps which are costly to produce, and the maps soon become out of date. The trails are commonly closed due to timber harvesting or other hazards that occur, so having easy ways to update the public of these hazards is critical. Furthermore, the GIS staff at GRCA have limited time to spend on these routine updates, so having a simple application for non-GIS savvy staff to update trail attribution and closure notifications is essential.

The Solution

Our team aimed to create an easy-to-use application for GRCA staff to select and change trail attribution to reflect current status and activity. Second, a Dashboard connected to the web map would be used to highlight any closures and hazards to users before they head out on the trail, combining written notifications currently on the GRCA website with map visualization of those hazards and other trail information.

Final Problem Statement

Ideal

A non GIS-saavy staff member at GRCA will be able to use an application to edit the status attributes of trails on a feature layer, and those attribute changes will reflect to symbology changes on a web map. Second, a dashboard highlighting areas of the forest that are closed, any reported hazards, and trails by activity.

Reality

There is currently a web map that is static, and the only way to change attributes on the web map would be to publish a new feature server from ArcMap or ArcGIS Pro and update the old feature server that way. Updates for trail closures and forest conditions are currently presented to the public as notices on the GRCA website, without any visual component.

Consequences

A GIS saavy staff member is required to make any change to the web map, a longer process and taking up their valuable time. Also, they stick with the same less intuitive notification system about forest conditions, without any visual tie to where those closures are located on the property.

Proposal

Two applications will be configured to create edits to the trail layer attributes, a Web AppBuilder application for desktop in-office edits, and an ArcGIS Field Maps application for in the field updates. A dashboard will be created to highlight trail status along with other useful widgets (list of all trails, list of closed trails etc.)
